Unlike conventional single-pass detectors, this project employs a **Sliding-Window Temporal Averaging Engine** that analyzes multiple overlapping audio segments to identify subtle synthesis artifacts left behind by modern neural vocoders.

# Overview

Modern voice cloning systems can generate speech nearly indistinguishable from human recordings. Traditional acoustic feature analysis is no longer sufficient for reliable detection.

This project addresses that challenge using a customized **ResNet-18** trained on forensic spectrogram representations to detect microscopic artifacts introduced during neural speech synthesis.

**Key Features**

- Modified ResNet-18 architecture
- Log-Mel Spectrogram input
- Sliding Window Temporal Averaging
- Binary Deepfake Classification
- Supports WAV, MP3, FLAC, and M4A
- GPU acceleration with PyTorch
- Batch inference pipeline

---

# Model Architecture

The detector is built upon a customized **ResNet-18** backbone specifically adapted for forensic audio analysis.

## Input Layer

The original RGB input layer was modified:

**Original**

- 3-channel RGB images

**Modified**

- 1-channel grayscale spectrograms

This allows the network to directly process Log-Mel spectrograms.

### Architecture

```text
Spectrogram
      │
Modified ResNet-18
      │
 Dropout (50%)
      │
 Linear Layer
      │
 Sigmoid
      │
Deepfake Probability
```

## Regularization

A 50% dropout layer helps prevent overfitting to:

- Microphone signatures
- Recording environments
- Background noise
- Dataset-specific artifacts

## Output

```text
0.00 → Genuine Human Speech
1.00 → AI Generated Speech
```

# Data Processing Pipeline

Raw audio cannot be directly processed by CNNs.

Each audio file undergoes the following preprocessing pipeline:

```text
Audio File
      │
Mono Conversion
      │
16 kHz Resampling
      │
Log-Mel Spectrogram
      │
Amplitude → dB
      │
ResNet-18
```

## Processing Steps

### 1. Mono Downmix

All stereo recordings are converted into mono.

### 2. Dynamic Resampling

Every file is resampled to:

```text
16,000 Hz
```

### 3. Log-Mel Spectrogram

Using **torchaudio**:

| Parameter | Value |
|-----------|------:|
| n_fft | 1024 |
| hop_length | 256 |
| f_min | 20 Hz |
| f_max | 8000 Hz |

### 4. Decibel Conversion

Power spectrograms are converted into the logarithmic dB scale to emphasize subtle synthesis artifacts.

# Inference Engine

A single prediction over an entire audio recording often misses localized synthesis artifacts.

Instead, this project uses **Sliding-Window Temporal Averaging**.

## Step 1

Split long recordings into overlapping windows.

```text
3 Seconds
50% Overlap
```

## Step 2

Each window is independently processed.

```text
Window 1 → Score

Window 2 → Score

Window 3 → Score

...
```

## Step 3

Average every prediction.

```text
Final Score

=

Mean(Window Scores)
```

### Why Averaging?

Peak extraction (`max()`) is highly sensitive to:

- Microphone pops
- Compression glitches
- Packet loss
- Clipping

Temporal averaging dramatically reduces false positives while preserving persistent synthesis artifacts.

# Known Limitations

Current limitations include:

- Heavy MP3 compression
- Smartphone DSP
- Noise suppression
- Aggressive equalization
- Extreme denoising

These processing pipelines may introduce artifacts that resemble neural vocoder fingerprints.
